# 多店铺订单功能说明

## 功能概述

购物车结算系统现在支持同时处理来自不同店铺的商品，为每个店铺创建独立的订单。

## 功能特性

### 1. 智能订单分组
- 系统自动按店铺ID对购物车商品进行分组
- 为每个店铺创建独立的订单
- 每个订单包含该店铺的所有商品

### 2. 多订单显示
- 订单确认页面显示所有创建的订单
- 每个订单显示独立的订单号、金额和支付宝订单号
- 清晰的订单分组展示

### 3. 独立支付
- 每个订单需要单独支付
- 支持逐一完成各订单的支付
- 每个订单有独立的支付链接

### 4. 用户体验优化
- 温馨提示说明多订单情况
- 清晰的操作指引
- 支持查看所有订单状态

## 技术实现

### CheckoutServlet 修改
```java
// 为每个店铺创建单独的订单
for (Map.Entry<Integer, List<CartItem>> entry : itemsByShop.entrySet()) {
```

### 订单确认页面支持
- 支持显示单个订单（向后兼容）
- 支持显示多个订单列表
- 动态生成支付按钮

## 使用流程

1. **添加商品到购物车** - 可以添加来自不同店铺的商品
2. **选择商品结算** - 选择要结算的商品（可以来自不同店铺）
3. **创建订单** - 系统自动为每个店铺创建独立订单
4. **确认订单** - 查看所有创建的订单详情
5. **逐一支付** - 为每个订单单独完成支付
6. **订单管理** - 在订单列表中查看和管理所有订单

## 优势

- **用户友好**: 一次结算，处理多店铺商品
- **订单清晰**: 每个店铺独立订单，便于管理
- **支付灵活**: 可以选择性支付部分订单
- **物流独立**: 每个店铺独立发货，互不影响

## 注意事项

- 每个订单有独立的30分钟支付时限
- 未支付的订单会自动取消
- 不同店铺的订单状态独立管理
- 评价系统按订单独立进行
