# 商家入驻和商铺管理功能测试指南

## 测试环境准备

1. 确保数据库连接正常
2. 确保项目已编译成功
3. 启动Web服务器

## 测试步骤

### 1. 用户注册/登录测试
1. 访问登录页面
2. 使用现有账号登录或注册新账号
3. 确认登录成功，能够访问个人中心

### 2. 商家入驻功能测试

#### 测试用例1：正常申请流程
1. 登录普通用户账号
2. 进入个人中心
3. 点击"商家入驻"菜单
4. 输入店铺名称（例如："测试店铺001"）
5. 点击"申请入驻"
6. 确认提交
7. **预期结果**：
   - 显示成功消息
   - 自动切换到商家角色
   - 可以在角色切换中看到商家选项
   - 导航栏出现"商铺管理"链接

#### 测试用例2：重复申请测试
1. 使用已经是商家的账号登录
2. 进入个人中心 -> 商家入驻
3. **预期结果**：
   - 显示"您已经是商家"的提示
   - 不显示申请表单

#### 测试用例3：店铺名称验证测试
1. 登录普通用户账号
2. 进入商家入驻页面
3. 测试以下输入：
   - 空白名称：应显示"请输入店铺名称"
   - 单字符："a"：应显示长度不足提示
   - 超长名称（51个字符）：应显示长度超限提示
   - 已存在的店铺名称：应显示名称重复提示

#### 测试用例4：未登录访问测试
1. 清除浏览器session
2. 直接访问 `/MerchantApplicationServlet`
3. **预期结果**：重定向到登录页面

### 3. 商铺管理功能测试 ⭐ 新增

#### 测试用例5：商铺管理页面访问
1. 使用商家账号登录
2. 在导航栏点击"商铺管理"
3. **预期结果**：
   - 成功进入商铺管理页面
   - 显示该商家的所有店铺
   - 每个店铺显示详细信息（ID、名称、创建时间等）

#### 测试用例6：非商家用户访问商铺管理
1. 使用普通用户账号登录
2. 直接访问 `/ShopManagementServlet`
3. **预期结果**：
   - 显示权限错误信息
   - 重定向到个人中心

#### 测试用例7：店铺名称修改功能
1. 商家用户进入商铺管理页面
2. 点击某个店铺的"编辑店铺"按钮
3. 修改店铺名称
4. 点击"保存修改"
5. **预期结果**：
   - 显示修改成功消息
   - 店铺名称已更新
   - 更新时间已刷新

#### 测试用例8：店铺名称修改验证
1. 在编辑表单中测试以下输入：
   - 空白名称：应显示验证错误
   - 单字符名称：应显示长度不足
   - 超长名称：应显示长度超限
   - 已存在的店铺名称：应显示名称重复错误

#### 测试用例9：无店铺状态测试
1. 创建新用户并申请成为商家但不创建店铺
2. 访问商铺管理页面
3. **预期结果**：
   - 显示"暂无店铺"的空状态
   - 提供前往申请入驻的链接

### 4. 导航栏动态显示测试 ⭐ 新增

#### 测试用例10：角色切换后导航栏更新
1. 使用拥有多个角色的用户登录
2. 在个人中心切换角色
3. **预期结果**：
   - 用户角色：显示"购物车"链接
   - 商家角色：显示"商铺管理"链接
   - 管理员角色：显示相应的管理链接

## 数据库验证

### 检查店铺创建
```sql
SELECT * FROM shop WHERE owner_id = [用户ID];
```

### 检查角色添加
```sql
SELECT ur.*, r.role_name 
FROM user_role ur 
JOIN role r ON ur.role_id = r.role_id 
WHERE ur.user_id = [用户ID];
```

### 检查店铺更新
```sql
SELECT * FROM shop WHERE shop_id = [店铺ID];
-- 检查 update_time 是否已更新
```

### 检查触发器执行
```sql
-- 应该能看到用户同时拥有用户角色(1)和商家角色(2)
SELECT COUNT(*) FROM user_role WHERE user_id = [用户ID] AND role_id = 2;
```

## 预期的数据库变化

### 申请成功后
1. `shop` 表新增一条记录：
   - `shop_name`: 用户输入的店铺名称
   - `owner_id`: 当前用户ID
   - `create_time`: 当前时间
   - `update_time`: 当前时间

2. `user_role` 表新增一条记录：
   - `user_id`: 当前用户ID
   - `role_id`: 2（商家角色）

### 店铺更新后 ⭐ 新增
1. `shop` 表对应记录更新：
   - `shop_name`: 新的店铺名称
   - `update_time`: 更新时间

## 常见问题排查

### 1. 申请失败
- 检查数据库连接
- 检查店铺名称是否重复
- 查看服务器日志

### 2. 角色未自动添加
- 检查数据库触发器是否存在
- 确认触发器权限正常

### 3. 页面显示异常
- 检查JSP页面语法
- 确认request属性设置正确

### 4. 商铺管理页面访问失败 ⭐ 新增
- 检查用户是否为商家角色
- 确认Servlet映射正确
- 查看权限验证逻辑

### 5. 店铺名称修改失败 ⭐ 新增
- 检查店铺所有权验证
- 确认名称唯一性检查
- 查看数据库更新日志

## 性能测试

1. 并发申请测试：多个用户同时申请不同店铺名称
2. 重复提交测试：快速多次点击提交按钮
3. 大量数据测试：创建大量店铺后的查询性能
4. 并发修改测试：多个商家同时修改店铺信息 ⭐ 新增

## 安全测试

1. SQL注入测试：在店铺名称中输入SQL语句
2. XSS测试：输入JavaScript代码
3. 权限测试：尝试访问其他用户的店铺信息
4. 跨用户操作测试：尝试修改其他用户的店铺 ⭐ 新增
5. 角色伪造测试：尝试以非商家身份访问商铺管理 ⭐ 新增

## UI/UX测试 ⭐ 新增

1. 响应式测试：在不同屏幕尺寸下测试页面显示
2. 交互测试：测试按钮点击、表单提交等交互
3. 错误提示测试：确认错误信息清晰易懂
4. 成功反馈测试：确认操作成功后的反馈明确
5. 空状态测试：测试无数据时的页面显示
